# 校准 rules：基于 glossary/current/

任务：

基于 glossary/current/ 中已经校准后的术语定义，重新整理和更新 rules/。

要求：

1. glossary/current/ 为唯一有效术语定义来源
2. 所有规则必须基于 current glossary 的定义
3. 如果旧 rules 与 current glossary 存在冲突：
   以 current glossary 为准

4. 保持规则：

   * 简洁
   * 可执行
   * 可观察
   * 可复盘

5. 删除：

   * 模糊表达
   * 情绪化语言
   * 不可执行规则
   * 重复规则

6. 合并语义重复规则
7. 优先保留：

   * 风险控制
   * 仓位管理
   * 市场环境
   * 情绪周期
   * 龙头地位

8. 不扩展新概念
9. 不新增原文不存在逻辑
10. 保持 rules 数量可控

输出要求：

* 更新 rules/ 下已有文件
* 如果发现规则高度重复：
  自动合并
* 如果规则依赖某个核心术语：
  明确标注关联术语

注意：

当前阶段目标不是构建最终交易系统，而是建立：
"术语定义一致" → "规则表达一致"
避免规则语义漂移。
