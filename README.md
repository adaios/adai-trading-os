# adai-trading-os

**交易课程整理 · 交易系统提炼 · 行为复盘** — AI 知识工程项目

将零散的交易课程转录文本，逐步加工为结构化的术语库、规则库和最终交易系统。

---

## 数据流

```
temp/ (原始下载) → raw/ (原始归档，不删除)
       │
       ▼
  cleaned/ (最低清洗) 
       │
       ├──→ glossary/ (逐课术语)
       │         │
       │         ▼
       │    glossary/current/ (融合术语库 = auto + manual)
       │         │
       ├──→ rules/ (逐课规则) ──→ 校准（基于 current）
       │                              │
       │                              ▼
       └──────────────────────── system/ (最终交易系统)
                                             ↑
                                    manual/glossary/ (人工修正，最高优先级)
```

## 目录结构

| 目录 | 用途 | 说明 |
|------|------|------|
| `temp/` | 从云盘下载的原始课程文本 | 按日期子目录存放，处理后保留合并 .md |
| `raw/` | 原始主题文本，**永久保留** | 每课一个 `.md` 文件，严禁删除 |
| `cleaned/` | 最低清洗后文本 | 仅删末尾SC念词名单，保留口语/闲聊/失败案例 |
| `glossary/` | 逐课术语提取 | 每课一个 `.glossary.md`，AI 自动提取 |
| `glossary/current/` | **融合后的统一术语库** | 唯一有效定义来源，auto + manual 融合生成 |
| `rules/` | 逐课规则提炼 | 每课一个 `.rules.md`，基于 glossary/current/ 校准 |
| `system/` | 最终交易系统 | 长期收敛后的核心系统 |
| `manual/glossary/` | 人工术语修正记录 | 高优先级，覆盖 AI 提取 |
| `manual/rules/` | 人工规则修正（预留） | — |
| `manual/system/` | 人工系统修正（预留） | — |
| `processed/` | 处理标记 | `.done` 文件标记已完成的课程 |
| `review/` | 交易复盘（预留） | — |
| `prompts/` | 流程定义 | process / fuse / calibrate 三个流程 |
| `scripts/` | 清理脚本 | cleanup.sh / cleanup.ps1 |

## 处理流程：4 阶段 9 步骤

| 阶段 | 步骤 | 内容 |
|:----:|:----:|:-----|
| **输入** | Step 1 | 扫描 temp/ → 合并到 raw/ |
| **产出** | Step 2 → Step 3 + 4 | cleaned / glossary / rules（后两步可并行） |
| **校准** | Step 5 → 6 → 7 | 融合 glossary / 校准 rules / 更新 system |
| **收尾** | Step 8 + 9 | .done 标记 / 清理 |

> 两个触发点：`temp/` 新增日期目录 或 `manual/glossary/` 新增修正记录。
>
> 详见 [CLAUDE.md](CLAUDE.md) 和 [prompts/process_all_courses.md](prompts/process_all_courses.md)。

## 核心工作原则

1. **绝不删除 raw/** 原始数据
2. 优先提炼明确规则，优先风险控制
3. 不做荐股，不主观发挥，不过度扩展系统复杂度
4. 保持系统简单、明确、可执行
5. 处理流程结束自动清理临时文件
6. 术语融合优先级：人工修正 > AI 自动提取
7. 季度收敛：去重、升级、剔除短期观点

