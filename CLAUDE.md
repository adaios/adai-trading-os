# adai-trading-os

这是一个用于交易课程整理、交易系统提炼、行为复盘的 AI 知识工程项目。

## 目标

- 对交易课程进行结构化整理
- 自动提炼交易术语
- 自动提炼交易规则
- 长期构建个人交易系统
- 降低信息噪音
- 提高交易系统一致性

---

## 目录结构

| 目录 | 用途 | 维护规则 |
|:----|:-----|:---------|
| `00-temp/` | 从云盘下载的课程原始文本（临时） | 流程结束后清理非标准输出 |
| `01-raw/` | 原始主题文本，每课一个 `.md` 文件 | **永久保留，不允许删除** |
| `02-cleaned/` | 最低限度清洗后的文本 | 可覆盖重做 |
| `03-glossary/` | 按课程归档的术语定义（每课一个 `.glossary.md`），含 `current/` 子目录 | 可覆盖重做，不追旧文件 |
| `04-rules/` | 按课程提炼的交易规则 | 可覆盖重做，融合后需校准 |
| `05-system/` | 最终交易系统，逐步积累，含 `archive/` 子目录 | 每课增量更新 |
| `06-processed/` | 流程标记（`.done` 文件） | 流程收尾时写入 |
| `07-manual/` | 人工修正记录（glossary / rules / system） | **任何时候不修改** |
| `08-review/` | 用户交易复盘 | 用户自主维护 |
| `09-scripts/` | 工具脚本 | 辅助工具 |
| `10-prompts/` | 提示词模板 | 辅助工具 |

### 目录修改权限

| 目录 | AI 能改吗 | 说明 |
|:-----|:---------|:------|
| `00-temp/` | ✅ 可读写 | 处理完后清理非标准输出，原始文稿保留 |

| `08-review/` | ❌ 不动 | 用户自主维护 |
| `09-scripts/` | ✅ 可写 | 辅助工具 |
| `10-prompts/` | ✅ 可写 | 提示词模板 |

## 工作原则

1. 永远不删除 01-raw/ 原始数据
2. 优先提炼明确规则
3. 优先风险控制
4. 不做荐股
5. 不主观发挥
6. 不过度扩展系统复杂度
7. 保持系统简单、明确、可执行
8. 流程结束后自动清理临时文件和残留
9. **禁止在 00-temp/ 下创建拆分/中间文件目录**，如处理过程中产生，流程结束后立即清理
10. **只改指定文件，除非在触发链内**：你指定改什么文件就只改什么文件。如需额外更新非指定文件（如融合时需同步 03-glossary/current/），必须说明依据并获得确认。

---

## 文本处理原则

- 仅做最低限度修剪：删除末尾SC念词名单、明显的设备调试空行
- **保留闲聊、个人色彩、口语表达**（用户日常复盘会反复琢磨的原话）
- 保留原文结构、交易逻辑、风险相关内容、案例、失败案例

---

## 术语提取原则

- 自动发现交易术语并归类
- 自动补充定义
- **不追回到旧文件**：后来课程补充的术语更新写入后来课程自己的文件，fusion 时合并到 03-glossary/current/
- **待确认区块搬迁**：glossary 文件（03-glossary/*.glossary.md）中不再列待确认。每课待确认统一放在 07-manual/glossary/YYYY-MM-DD_主题.md 末尾，格式为 | 词汇 | 原因 | 表格。待确认列 AI 不确定的词汇：黑话/未知缩写/通过上下文推测但无原文定义的词汇，只要不是100%确定就列出来供确认。不做判断性质疑。

---

## 规则提炼原则

- 将经验表达转换为规则表达，尽量结构化
- **保留原文金句作为备注**（用户日常复盘的素材）
- 不创造原文不存在的规则
- **术语表达约定**：系统内优先使用标准术语，特色词汇以括号附注在后。如：暴力突破（暴力特噗）、减仓一半（卤煮/放飞）

## 核心流程

整个项目只有一条数据流链，上下游依赖关系固定：

```
01-raw → 02-cleaned → 03-glossary → 04-rules → 05-system
                             ↑            ↑
                        07-manual/   07-manual/
                        glossary     rules → 07-manual/system
                        审核修正    审核修正        直接覆盖
```

### 依赖链（单向，不可逆）

```
03-glossary（定义源头）
    ↓ 依赖
04-rules（规则层，引用 03-glossary 术语）
    ↓ 依赖
05-system（系统层，基于 04-rules 和系统贡献区块组装）
```

上游变更会破坏下游一致性，但重建是**按需触发的**（非自动），由用户确认后执行。

---

## 全流程概览

整个项目的运转分三个层次，逐层递进：

```
第一层：单课处理（每课独立跑完 Step 1→5）
              ↓
第二层：批量收敛（多课积累后融合术语、校准规则、整合系统）
              ↓
第三层：人工审核（07-manual 修正 → 按需重建下游）
```

---

## 第一层：单课处理流程

> 一课一课处理。从最早课程开始，按日期顺序推进。

```
Step 1  raw  ──→  Step 2  cleaned  ──→  Step 3  glossary
                                              │
                                              ↓
                                         Fusion（术语归入 14 分类）
                                              │
                                              ↓
                                         Step 4  rules
                                              │
                                              ↓
                                         Step 5  系统更新
```

### Step 1：raw（文稿准备）

合并 `00-temp/` 下的文稿为 `01-raw/YYYY-MM-DD_主题.md`（如已有 `01-raw/` 文件则跳过）。
多部分文稿按顺序拼接，删除文件分隔标记和 AI 标注，首行写标题。

**产出：** `01-raw/YYYY-MM-DD_主题.md`

### Step 2：cleaned（清洗）

仅做最低限度修剪：删除末尾SC念词名单、明显的设备调试空行。
其余内容（闲聊、口语、个人色彩）全部保留。

**产出：** `02-cleaned/YYYY-MM-DD_主题.cleaned.md`

### Step 3：glossary（术语提取）

提取本课的交易术语、战法、口诀，写入 `03-glossary/*.glossary.md`。

**每个 glossary 文件结构：**
```
## 术语名称
- **定义**：xxx
- **特征**：xxx（可选）
- **市场意义**：xxx
- **常见误区**：xxx（可选）

## 本课系统贡献

记录该课对交易系统的增量贡献，区分三个层面：

**框架层：** 提供了系统架构/思维模型（如 4231+四不=最小系统骨架）
**工具层：** 提供了具体战法/指标（如 三大指标组合 KDJ+BBI+MACD）
**认知层：** 提供了心态/理念（如 "怕就会输一辈子"）
```

> **注意：** glossary 文件中不再包含"## 待确认"区块。不确定的术语统一写入 `07-manual/glossary/YYYY-MM-DD_主题.md` 末尾。

**产出：** `03-glossary/YYYY-MM-DD_主题.glossary.md`

### Fusion：术语融合（每课 Step 3 后立即执行）

将本课新术语归入 `03-glossary/current/glossary.md`，按 **14 个交易系统分类** 组织。每个术语只出现一次，不跨类重复。

| # | 分类 | 包含范围 |
|:-:|:-----|:---------|
| 1 | 交易系统总纲 | 六步闭环、执行口诀、正确vs错误交易、执行三段论、复盘五步法 |
| 2 | A股市场底层逻辑 | 国家意志、政策周期、十年周期、168万亿、反内卷、China New Story、顺周期、踏空鉴别、信息噪音鉴别 |
| 3 | 择时体系 | 择时三要素、活跃市值/0AMV、入场分级、2.3%离场法则、回测原则 |
| 4 | 选股与配置 | 选股逻辑链、曼城阵容、六大赛道、机构资金排序、填坑出坑、坑不能做 |
| 5 | N型结构与节奏 | N型结构、三层节奏体系、吸呼感知法、回撤vs反弹、机构vs游资呼吸区别 |
| 6 | 量价与筹码 | 缩量真实放量可作假、3/4阴量线、顶部无量、顶部大风车、筹码纯度、跟风盘成本交换 |
| 7 | B1买点体系 | B1/B2/B3/SB1、买入三问、回调一半法则、接力图形、合并同类项 |
| 8 | 入场方式与战法 | 暴力特噗、深水炸弹、单针下20、补票战法、三波理论循环 |
| 9 | 止损体系 | 只输一根K线、止损三级别、止损三原则、压力测试 |
| 10 | 止盈与收队 | BBI线上减仓一半、见顶信号、收队规则 |
| 11 | 仓位管理 | 四不原则、仓位纪律、两枪原则、短线不加仓 |
| 12 | 市场参与者认知 | 机构vs游资vs量化、跟庄三条件、公募行为、麒麟会博弈 |
| 13 | 交易心理与纪律 | 核心认知、市场跟随原则、常见错误 |
| 14 | 特色词汇与口诀 | 黑话表、所有口诀 |

**产出：** `03-glossary/current/glossary.md`

### Step 4：rules（规则提炼）

基于本课内容提炼规则，写入 `04-rules/*.rules.md`。
每条规则标注来源金句，不创造原文不存在的规则。

**产出：** `04-rules/YYYY-MM-DD_主题.rules.md`

### Step 5：系统更新

根据当前所有已处理课程（含本课），更新 `05-system/trading-system.md`：

1. **如果 `07-manual/system/trading-system.md` 有内容**，以 manual 结构为准（少妇战法六步为主干），仅在其框架下填充新课程的贡献记录
2. **如果 `07-manual/system/trading-system.md` 无内容**，遍历 `03-glossary/*.glossary.md` 的 `## 本课系统贡献` 区块，按框架层/工具层/认知层分类汇总，去重（后期课程版本优先）
3. 每课贡献记录追加到附录，不干扰正文结构
4. 旧版本归档到 `05-system/archive/`

**产出：** `05-system/trading-system.md`

### 单课收尾

每课 Step 1→5 完成后：

1. 写入 `06-processed/YYYY-MM-DD_主题.done` 标记完成
2. 清理 `00-temp/` 下该课已处理的原始文稿（非标准输出删除，原始文件保留）
3. 等待用户确认或写 `07-manual/` 修正后，再处理下一课

## 第二层：批量收敛（多课积累后执行）

当累积到一定数量课程（如一个季度 / 一个主题阶段），执行一次收敛：

### Phase A：融合术语重建

```
遍历所有已处理课程的 glossary
       ↓
按 14 分类重新组装 03-glossary/current/glossary.md
       ↓
应用 07-manual/glossary/*.md 中的修正覆盖 AI 定义
```

### Phase B：规则校准

```
检查 04-rules/*.rules.md 之间的冲突
       ↓
如有 07-manual/rules/*.md 修正，覆盖对应规则
       ↓
合并重复规则，保留后期课程版本
```

### Phase C：系统重建

```
基于融合后的术语表 + 校准后的规则 + 07-manual/system
       ↓
重建 05-system/trading-system.md
       ↓
旧版本归档到 05-system/archive/
```

## 第三层：人工审核与修正

### 审核方式

你在任意环节对产出不满意，写入 `07-manual/` 对应文件：

| 位置 | 修正内容 | 影响范围 |
|:----|:---------|:---------|
| `07-manual/glossary/YYYY-MM-DD_主题.md` | 术语定义修正 | 融合时覆盖 AI 定义 |
| `07-manual/rules/YYYY-MM-DD_主题.md` | 规则表述修正 | 规则校准 |
| `07-manual/system/trading-system.md` | 系统结构修正 | 系统重建时直接采用 |

### AI 行为约束

`07-manual/` 有修正意见时，AI **只确认收到，不直接修改对应的 `03-glossary/` 或 `04-rules/` 产出文件**。修正意见在融合/校准流程中自动覆盖 AI 自动定义。

### 重建触发链（按需触发，由用户确认）

```
07-manual/glossary/ 变更 → 03-glossary/current/ 重融合 → 04-rules 重校准 → 05-system 重建
07-manual/rules/ 变更 → 04-rules 重校准 → 05-system 重建
07-manual/system/ 变更 → 05-system 直接覆盖（不做自动化处理）
```

### 手册状态约定

| 状态 | 含义 | 处理方式 |
|:----|:-----|:---------|
| 文件不存在 | 该课尚未审核 | glossary/rules 正常产出，参与 05-system 组装 |
| 文件有内容 | 有修正意见 | fusion/校准 时覆盖 AI 自动定义 |
| 文件为空 | 已审核无需修正 | 维持 AI 自动定义 |

---

## 总处理顺序

1. **按课程日期升序处理**（最早课程先处理）
2. 课程体系有演化过程，后期内容可能修正前期规则
3. **处理完一课，用户确认后，再走下一课**
4. 批量收敛（第二层）在用户指定时执行，不定期

---

## 最终目标

形成一个长期稳定、可执行、可复盘的个人交易系统。
